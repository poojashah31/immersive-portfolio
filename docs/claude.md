# Fantasy Portfolio Development Guide

This repository contains a cinematic, story-driven fantasy portfolio.

The objective is to create an immersive experience rather than a traditional portfolio website.

Always prioritize atmosphere, storytelling, maintainability and modular architecture over speed of implementation.

---

## Read Before Every Task

Before making any changes, always read:

docs/PROJECT_CONTEXT.md

docs/SETUP_SPEC.md

docs/DEVELOPMENT_PLAN.md

docs/REVIEW_LOOP.md

Also review the relevant files inside the references folder for the current milestone.

---

# Visual Direction

The project should feel:

- mysterious
- magical
- elegant
- warm
- cinematic
- immersive
- medieval fantasy

Avoid:

- modern UI aesthetics
- cyberpunk
- excessive bloom
- visual clutter
- unnecessary effects

References are artistic inspiration only.

Never recreate or copy artwork directly.

Instead analyze:

- atmosphere
- composition
- lighting
- pacing
- camera movement
- storytelling
- color palette

Then create an original implementation.

---

# Development Philosophy

Work in milestones.

Never implement multiple milestones together.

Complete one milestone before beginning the next.

Every milestone should be:

- modular
- reusable
- maintainable
- polished
- production ready

If architecture can be improved, explain why before implementing.

---

# Review Loop

For every task:

Implement

↓

Run

↓

Review

↓

Compare with acceptance criteria

↓

Refactor if necessary

↓

Repeat

↓

Stop

Never continue automatically to the next milestone.

Always wait for confirmation.

---

# Coding Standards

Use:

- React
- Vite
- React Three Fiber
- Drei
- GSAP

Use functional React components.

Keep files focused and modular.

Avoid duplicated logic.

Prefer reusable abstractions.

Use descriptive naming.

Separate rendering, animation and scene logic.

Avoid hardcoded values whenever practical.

Keep components reasonably small.

---

# Asset Guidelines

Current assets are placeholders.

Do not replace them unless requested.

Use the provided assets whenever possible.

If an asset has loading issues, fix the loading process instead of replacing the asset.

Future optimization (GLB conversion, texture compression, asset optimization) can be suggested but should not block development.

---

# References

The references folder contains inspiration only.

Never reproduce any artwork.

Extract the artistic direction instead.

---

# Deliverables

After completing any task provide:

- Summary
- Files Created
- Files Modified
- Architectural Decisions
- Remaining Improvements

Then stop.

Never begin another milestone automatically.
